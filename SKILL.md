# Trip Destination Card

> 将目的地信息转化为精美的"电子杂志 × 电子墨水"风格卡片，一键生成可翻页可导出的行程方案。

## 这个 Skill 做什么

生成一份**单文件 HTML**的行程目的地展示卡片，风格为"电子杂志 × 电子墨水"：

- **衬线标题（Noto Serif SC）+ 非衬线正文（Noto Sans SC）+ 等宽数据（JetBrains Mono）**
- **暖色纸质感背景 + 细腻边框**
- **单酒店：左信息（交通/天气/酒店/优缺点）右图片；双酒店：左中右三栏（交通天气 | 酒店A+图片 | 酒店B+图片）**
- **横向翻页（键盘 ← →、滚轮、触屏）+ 纵向长图双模式，ESC 缩略索引**
- **一键导出为长图 PNG（html-to-image）**

美学锚点：像 *Monocle* 杂志的旅行特辑，优雅、安静、有质感。

## 何时使用

**合适的场景**：
- 给老板做行程方案备选展示（3-4 个目的地对比）
- 商务出差提案
- 高管 retreat / 闭关选址方案
- 需要"一眼看清、方便对比"的目的地卡片

**不合适的场景**：
- 详细旅游攻略（信息维度不够）
- 多人协作编辑（这是静态 HTML）

## 信息维度

每个目的地的卡片包含以下信息（用户提供文字和图片，Skill 负责填入）：

| 维度 | 说明 | 示例 |
|---|---|---|
| **目的地名称** | 城市/地区名 | 乌鲁木齐、伊宁、拉萨 |
| **交通** | 航班/车次、时长、时刻、机型、特色说明 | 北京直飞4h，大飞机无wifi |
| **天气** | 温度范围、特殊气候提示 | 7月20-30℃，温差大，紫外线强 |
| **酒店** | 名称、开业年份、房型、面积、价格、特色。一个目的地可有 1-2 个酒店 | 希尔顿/洲际/瑞吉+松赞 |
| **优点** | 3-5 条推荐理由（双酒店时每店独立） | 位于市区，配套完善 |
| **缺点** | 2-3 条注意事项（双酒店时每店独立） | 少一些"避世感" |
| **图片** | 酒店或目的地照片。单酒店 1-3 张，双酒店每店 1 张 | 用户提供的图片文件或 URL |

## 信息标准化规范

无论用户以什么格式提供原始信息（PDF、聊天消息、语音转写、表格截图），
Skill 都将其标准化为以下字段后再填入模板。

### 每个目的地的标准字段

| # | 字段 | 格式规范 | 示例 |
|---|---|---|---|
| 1 | **目的地名称** | 城市名，可选副标题（区域/特色） | 乌鲁木齐 · 北疆·天山北麓 |
| 2 | **交通·总述** | 出发地 + 交通方式 + 时长 | 北京直飞 4h |
| 3 | **交通·去程** | 日期 星期 出发时间–到达时间 出发地→目的地 | 7/2 周四 20:55–01:15 首都→乌市 |
| 4 | **交通·返程** | 日期 星期 出发时间–到达时间 出发地→目的地 | 7/5 周日 14:35–18:10 乌市→首都 |
| 5 | **交通·备注** | 机型、WiFi、中转等 | 大飞机，无 WiFi |
| 6 | **天气·温度** | 月份 + 温度范围 | 7月 20–30℃ |
| 7 | **天气·提示** | 特殊气候注意事项 | 温差大，紫外线强 |
| 8 | **酒店·名称** | 全称 | 乌鲁木齐希尔顿酒店 |
| 9 | **酒店·年份** | 开业年份 | 2015 |
| 10 | **酒店·房型** | 具体房型名称 | 希尔顿套房 |
| 11 | **酒店·面积** | 数字 + ㎡ | 110㎡ |
| 12 | **酒店·价格** | 数字 + 元/晚（不含千分位逗号） | 3200元/晚 |
| 13 | **酒店·露台** | 有/无 | 无露台 |
| 14 | **酒店·吸烟** | 可吸烟/禁烟/室内可吸烟 | 室内禁烟 |
| 15 | **优点** | 3-5 条，每条一句话 | 位于市区，周边配套完善 |
| 16 | **缺点/注意** | 2-3 条，每条一句话 | 少一些"避世感" |
| 17 | **图片** | 标注所属目的地即可，横版/竖版 Skill 自动识别 | 乌鲁木齐-酒店外观.png |

- 第 8-14 项（酒店相关）如果目的地有**两个酒店**，则分别提供两套。
- 图片按横版（宽>高）和竖版（高>宽）分类提供。

## 工作流

### Step 1 · 返回问题清单（必须先做）

**在用户提供素材之前或素材不完整时**，先回复以下清单让用户对照收集。
不要在信息不完整时直接开始构建。

```
📋 请按以下清单提供每个目的地的信息：

【目的地名称】城市名（如"乌鲁木齐"）

【交通】
  · 总述：（如"北京直飞 4h"）
  · 去程：日期 + 星期 + 时间 + 起止地（如"7/2 周四 20:55–01:15 首都→乌市"）
  · 返程：日期 + 星期 + 时间 + 起止地
  · 备注：机型 / WiFi / 中转等

【天气】
  · 温度：（如"7月 20–30℃"）
  · 提示：（如"温差大，紫外线强"）

【酒店】（每个目的地 1-2 个酒店，每个酒店分别填写）
  · 名称 / 开业年份 / 房型 / 面积 / 价格
  · 露台：有 / 无
  · 吸烟：可吸烟 / 禁烟 / 室内可吸烟

【优点】（每个酒店 3-5 条）
【注意】（每个酒店 2-3 条）

【图片】每个目的地需提供：
  · 2 张酒店图片（双酒店时每个酒店各 2 张）
  · 3 张风景/目的地图片
  · 标注属于哪个目的地即可，横版/竖版自动识别
```

用户可以选择：
- 逐项填写清单
- 直接发原始材料（PDF、聊天记录等），由 Skill 提取并标准化
- 混合方式：发材料 + 补充清单中缺失的字段

### Step 2 · 标准化与填入模板

1. 将用户提供的所有信息标准化为上述 17 个字段
2. 复制 `assets/template.html` 到工作目录
3. 保持 `<style>` 块完整（这是设计系统）
4. 填入内容：
   - 单酒店目的地：使用 Card 1 / Card 2 结构
   - 双酒店目的地：使用 Card 3 三栏结构
5. 图片处理：
   - 本地图片：转为 base64 嵌入（推荐，离线可用）
   - 远程 URL：保留 URL + `onerror` 回退
   - 横版图片放入 `.img-col-landscape`，竖版放入 `.img-col-portrait`
   - 双酒店每店图片放入 `.hotel-imgs`

### Step 3 · 检查与交付

- 图片完整性：确认所有图片可加载
- 信息准确性：逐卡核对，特别注意酒店名、价格、航班时间
- 翻页流畅：← → 键和滚轮翻页正常
- 长图模式：切换后排版正常
- 导出测试：点击 📥 导出确认截图完整

## 展示模式

### 翻页模式（默认）
- 键盘 ← → 翻页
- 触屏左右滑动
- 鼠标滚轮翻页
- ESC 键显示索引总览

### 长图模式
- 所有卡片纵向排列
- 适合截图分享、打印、嵌入文档
- 右上角按钮切换

### 导出图片
- 使用 html-to-image 库将长图模式导出为 PNG
- 一键下载

## 主题色

默认暖色电子墨水调色板（`assets/template.html` 中的 `:root` 变量）：

```css
--paper: #F6F2EA;       /* 纸色背景 */
--ink: #2C2416;          /* 墨色文字 */
--ink-soft: #6B5E4F;     /* 次级文字 */
--accent: #8B6C42;       /* 强调色 */
--line: rgba(44,36,22,.12);
```

如需换色，只改动 `:root` 变量，不要修改组件样式。
